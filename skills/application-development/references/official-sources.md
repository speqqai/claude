# Official Sources

Use these sources in priority order. Keep the set small and official.

## Core framework sources

1. Next.js docs
- https://nextjs.org/docs
- Use for routing, rendering strategy, data fetching, caching, Server/Client Components, deployment guidance.

2. React docs
- https://react.dev/
- Use for state/effect patterns, component architecture, and modern React guidance.

3. TypeScript docs
- https://www.typescriptlang.org/docs/
- Use for strict typing strategy, type narrowing, module boundaries, and project configuration.

4. Node.js docs
- https://nodejs.org/docs/latest/api/
- Use for runtime behavior, process model, streams, timers, and server-side primitives.

## Quality and standards sources

5. MDN Accessibility (Core)
- https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Accessibility
- Use for semantic HTML, forms, keyboard and screen reader support.

6. W3C WCAG overview
- https://www.w3.org/WAI/standards-guidelines/wcag/
- Use for normative accessibility criteria and conformance references.

7. Web.dev performance learning path
- https://web.dev/learn/performance
- Use for page speed, rendering costs, loading strategy, and optimization priorities.

8. Web Vitals
- https://web.dev/articles/vitals
- Use for measuring and prioritizing real-user performance quality.

## Next.js styling-specific sources

9. Next.js styling docs
- https://nextjs.org/docs/app/building-your-application/styling
- Use for CSS Modules, global styles, and framework-native styling patterns.

10. MDN CSS custom properties
- https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_cascading_variables/Using_CSS_custom_properties
- Use for centralized tokens and shared design variables.

## Source-selection rule

When sources conflict, prefer this order:
1. Framework docs for framework behavior.
2. Language/runtime docs for language/runtime behavior.
3. Standards docs (W3C) for accessibility criteria.
4. MDN/Web.dev for implementation guidance.
